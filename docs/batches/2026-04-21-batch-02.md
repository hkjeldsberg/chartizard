### What to tune for Batch 3

1. **Large-dataset charts benefit from data-shaped anchors, not mark-shaped anchors.** Sub-agent A3's word-cloud and B2's network both anchored concepts (size-encoding, community, layout) rather than specific marks. That reads better in Explain mode than "here's hex #43".
2. **Keep flagging deps proactively.** Sub-agent B2 flagged `d3-chord` clearly; Sub-agent B1 correctly noted `d3-hierarchy` was already in-tree. The habit is sticking.
3. **Consider a shared `datasets/` module for the two-chart "compare the story" pattern.** Batch 1 A3/A4 (scatter↔bubble, box↔violin) and Batch 2 A2/A3 (choropleth↔cartogram, word-cloud's stopwords tie-in) all used the same dataset twice. At 4 duplications the DRY pressure starts to win. Evaluate in Batch 3 — if another pair surfaces, extract `src/content/datasets/`.
4. **Folder naming quirk:** `src/charts/control-chart/ControlChartChart.tsx` (double "Chart") reads awkwardly. Consider a contract rule: *if the slug already ends in `-chart`, drop the second Chart suffix*. Or normalize to `ControlChart.tsx`. Document in contract §1 before Batch 3.
