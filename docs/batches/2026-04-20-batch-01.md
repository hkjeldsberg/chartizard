### What to tune for Batch 2

1. **Contract ambiguity: anchor hit-rect clamping.** Two sub-agents (A3, A5) asked whether anchor rects should clamp to chart bounds. Contract ¬ß8 mentions "clamp `rect` to `{ x: 0, y: 0, width: iw, height: ih }` when in doubt" but should make this a stronger statement: **clamp always unless the anchor is intentionally outside the plot area** (e.g. y-axis label in the left margin).
2. **Contract ambiguity: outlier/conditional anchors.** Slot A4 dropped box-plot's `outlier` anchor because the 1.5√óIQR rule didn't always fire. Contract should say: **anchors must always render ‚Äî don't add a selector that depends on runtime data to fire.** If you want to call out "where outliers would appear", use a concept anchor (e.g. "outlier-rule") rather than one pinned to a specific outlier circle.
3. **Dep flagging worked.** B5's "list every dep flag in your report" instruction produced clean, complete dep additions. Keep this pattern.
4. **Slot size is right.** 2 charts per slot is the sweet spot. Wave A averaged ~5 min, Wave B ~5 min. Going to 3 per slot would start producing rushed output on chart #3.
5. **Parallel-slot awareness.** Several sub-agents saw partially-written files from other slots mid-run and reported them as "pre-existing errors". Consider noting in the brief: *"other sub-agents are writing files concurrently ‚Äî if you see TS errors in `heatmap/` or `sankey/` etc. during your work, those are in-flight, not yours to fix."*

### Contract updates to apply before Batch 2

Apply these to `docs/chart-author-contract.md`:
- ¬ß8 "Common pitfalls": strengthen the rect-clamping rule as noted above.
