Mark gain margin (dB-distance from 0 dB at the phase-crossover frequency) and phase margin (degree-distance from −180° at the gain-crossover frequency). Story: Hendrik Bode, Bell Labs 1940s. Story: the chart that answers "does this series remember itself?" A spike at lag 12 on monthly data reveals annual seasonality; a slow decay means the series is non-stationary. Story: ACF gives you the *total* relationship at each lag; PACF gives the *direct* relationship. The reading rule: AR(p) → PACF cuts off after lag p, ACF tails off; MA(q) → ACF cuts off after lag q, PACF tails off. A high-AUC model can be terribly mis-calibrated; a lower-AUC one perfectly calibrated. Name Platt scaling and isotonic regression as the two canonical post-hoc calibration fixes. |
| A4   | `pp-plot`                      | P-P Plot                            | distribution | statistics                  | S    | Scatter of (F̂(x_(i)), F(x_(i))) pairs — empirical CDF value on x, theoretical CDF value on y — for each ordered observation. ~40 points. 45° diagonal = perfect fit to the reference distribution. Both answer "does this sample match this distribution?" — QQ plots quantiles (linear in the variable's units), PP plots cumulative probabilities (linear in probability units). PP is more sensitive in the centre of the distribution; QQ is more sensitive in the tails. Name Wilk & Gnanadesikan 1968 (*Probability Plotting Methods*) as the modern formulation of both. |

## Notes

- **Deps likely needed:** none. UML/BPMN/DFD/CFG/VSM all use primitive SVG shapes (rects, circles, lines, polygons) + `<marker>` arrowheads — pattern established in Batch 7 (UML class/ERD/sequence, FSM, PERT). Phylogenetic tree reuses the dendrogram-family rectangular-layout pattern from Batch 2. Cladogram is a phylogenetic-tree with uniform branch length. SHAP summary reuses the beeswarm pattern from Batch 4. Bode plot reuses stacked log-scale plots — `scaleLog` already in use.
- **Known trickiness:**
  - **Bode plot** has TWO panels stacked vertically sharing the same x-axis. Top panel needs `scaleLog` on x and linear dB on y; bottom panel `scaleLog` on x and degrees on y. Guard the log-scale domain — `scaleLog` throws on 0; start at ω=0.01 or 0.1.
  - **Pole-zero plot** needs clear x/z marker distinction (× = pole, ○ = zero). Render with text glyphs or path marks, not emoji.
  - **ACF/PACF** — the ±1.96/√n confidence band for n=100 is ±0.196, so show ~±0.2 dashed horizontal lines. Stems are vertical lines with dots at the tips.
  - **PDP** needs an average curve AND a background hint of ICE lines (thin, low-opacity). The ICE overlay is part of the chart's modern teaching story; omit and it looks like just a scatter-with-line.
  - **SHAP summary** — beeswarm per feature requires jittering in y within each feature row. Each row needs ~40-60 dots. Colour encoding is feature value (blue-red diverging), NOT SHAP value.
  - **Calibration plot** — the "bin" semantics: dots sit at the mean predicted probability within each bin horizontally, the observed positive rate vertically. Include a reference 45° line.
  - **PP-plot** vs QQ-plot — PP has (0,0) to (1,1) axis bounds (both are probabilities), while QQ has variable-scale axes (theoretical quantiles). Don't accidentally render a QQ plot.
  - **Argand diagram** needs a drawn vector from origin to the plotted point with an arrowhead. Include the conjugate to make the diagram's geometry-of-complex-conjugation visible.
  - **Feynman diagram** — the THREE line types matter:
    - Solid line with arrow = fermion (arrow direction encodes particle vs antiparticle: electron ↘, positron ↖ in time-space).
    - Wavy line = photon (massless boson).
    - For a simple tree-level diagram, stick to fermion + photon; skip gluon loops (would need spiral path) and W/Z (dashed).
  - **UML Activity** — fork/join bars are thick black horizontal rectangles (not thin lines). Guards on decision-outgoing edges go in square brackets `[credit_ok]`.
  - **UML State** — composite (nested) states render as a large rounded-rectangle containing smaller rounded-rectangles, with the composite's label in the top-left corner.
  - **DFD notation** — Yourdon uses circles for processes; Gane-Sarson uses rounded rectangles. Pick Yourdon for consistency with textbook usage; mention Gane-Sarson in the narrative.
  - **CFG** — basic blocks are usually rendered as rectangles with multi-line content (first statement + "..." + last statement). Edges are labelled T/F on conditional branches; back-edges (loops) need clearly curved/routed paths to not overlap forward edges.
  - **BPMN** gateways — X in a diamond = exclusive, + in a diamond = parallel, O in a diamond = inclusive. Start/intermediate/end event circles differ only in stroke thickness (thin / double / thick).
  - **Value Stream Map** has more visual complexity per unit area than any other chart in this batch. Data tiles under each process box with 4 rows (C/T, C/O, Uptime, Operators) pack a lot of numbers. Inventory triangles between processes need a number inside (queue size). The sawtooth timeline at the bottom needs distinct value-add vs non-value-add y-levels. This is a `tileSize: W` for a reason.
  - **Phylogenetic tree & cladogram** — rectangular dendrogram-family layout. `d3-hierarchy`'s `cluster()` layout gives coordinates; join internal node to children with two line segments (vertical then horizontal) for "rectangular" phylogram style.
  - **RACI matrix** — use the proper filled-vs-outlined distinction for A vs R/C/I. Colour is insufficient (fails colour-only encoding rule). Consider a square glyph: ■ (filled) for A, □ (outlined) for R, ◇ for C, · for I.
  - **SIPOC diagram** — five columns of unequal width. Process column typically takes 30-40%; Suppliers/Customers 15% each; Inputs/Outputs 15% each. Column headers need a visual divider (horizontal rule below).
- **Slot routing:**
  - **Wave A** — all Sonnet. Smaller charts, single-panel (Bode has two panels but they share an axis and the pattern is well-established).
  - **Wave B** — Opus for B1 (UML activity + state: swimlanes, fork/join bars, composite states — layout coordination with N>6 distinct shape primitives and multiple relationship types), B2 (DFD + CFG: Yourdon circles + data stores + back-edges + labelled edges — mixed shape primitives + careful edge routing), B3 (BPMN + VSM: swimlane pool + gateway diamonds + event circles for BPMN; supplier/input/process/output/customer columns + timeline + data tiles for VSM — each is a distinct layout discipline).
