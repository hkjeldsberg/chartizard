## Notes

- **Deps likely needed:** none. All charts use existing Visx + d3 stack. Tile-map and hexbin-map need US-state coordinates (hand-roll — no real GeoJSON needed at 50 tiles). Flow-map can use hand-rolled lat/lng to screen-space projection (don't pull in d3-geo for 12 curves). Smith chart is pure analytic geometry (circles of the form `(x-a)² + y² = r²`).
- **Known trickiness:**
  - **Bivariate choropleth:** 3×3 colour matrix means 9 distinct fills. Use two-hue diverging schemes (low-low = light neutral, high-high = saturated blue-purple). Include the 3×3 legend inline.
  - **Flow-map:** curves between origin and destination need a Bezier curvature proportional to distance, otherwise long-distance arcs look straight. Hand-roll the control-point math.
  - **Parallel sets:** like Sankey but each stage is a set of categories; ribbons carry category×category counts. Order of categories within each axis matters for minimising crossings.
  - **Cumulative-flow-diagram:** stacked-area on a time axis, but the bands are *state-of-a-ticket* not quantities. Use the `AreaStack` pattern but explicitly order the stack: Done (bottom, dark) → In Review → In Progress → To-Do (top, light).
  - **Smith chart:** the constant-resistance circles all pass through (1, 0); the constant-reactance arcs all emanate from (1, 0). That shared point IS the chart's design — don't try to plot the circles with arbitrary centres.
- **Slot routing:** Wave B has 4 `L` and 2 `W` slots — route B1 (hexbin-map L), B2 (flow-map + bivariate-choropleth both L), B3 (parallel-sets + swimlane both W), and B4 (mind-map + CFD both L) to Opus. B5 (phase-diagram + smith-chart, both M) to Sonnet.
