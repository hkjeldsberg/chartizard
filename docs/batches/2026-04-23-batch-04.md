## Theme for Batch 4

This batch leans into **statistics workbench** + **time-series density** charts. Every Wave A slot is a variant of "show the shape of a distribution" (jitter → strip → beeswarm → sina → stem-and-leaf → radial histogram). Every Wave B slot is either a **grid-of-charts** (small multiples, correlation matrix, SPLOM) or a **dense time-series** view (horizon, calendar, spaghetti, run-chart). This gives us five new pair-charts where two charts share a dataset and the narrative is "why this encoding over that one":

- jitter / strip / beeswarm / sina — 1D distribution alternatives
- horizon / calendar / spaghetti — time-series density variants
- correlation-matrix / scatter-plot-matrix — same data, two encodings
- contour / rose-diagram — angular vs Cartesian density
- radial-histogram / rose-diagram — both angular, different domains

The DRY pressure for a shared `src/content/datasets/` module keeps rising. Still deferred to Batch 5 pre-flight — evaluate after this batch lands.

## Notes

- **Deps likely needed:** none. All 20 charts implement with the existing Visx + d3-shape / d3-array / d3-hierarchy stack. The beeswarm/sina packing uses a force-free deterministic algorithm (sort + greedy y-offset); no `d3-force`.
- **Known trickiness:**
  - **Beeswarm packing**: naïve force simulation is non-deterministic. Use deterministic "rank + greedy" packing (sort by value, walk each point to the lowest non-colliding offset). See `src/charts/violin/ViolinChart.tsx` pattern for seeded jitter.
  - **Horizon chart**: band stacking + colour mapping needs explicit mirror-fold math — bands above baseline stack upward, bands below fold to positive then stack. Document clearly in the metadata.
  - **Calendar**: week-column / day-row layout. Use a deterministic start date (`2024-01-01`, 52 weeks). Skip real date libs — hand-roll day-of-week math.
  - **SPLOM**: 4×4 grid of scatters requires a nested render with per-pair scales. `tileSize: "L"` gives enough room at 800×800.
  - **Radial-histogram + Rose-diagram**: two polar charts in the same batch. Don't share code — the stories are different (radial histogram = cyclical x-axis, rose diagram = frequency over compass). Separate implementations.
- **Slot routing:** Route Wave B slots with heavy layout work (B1 small-multiples, B3 SPLOM, B2 calendar) to `general-purpose` Opus; Wave A is all Sonnet.
