## Notes

- **Deps likely needed:** none. All charts use SVG primitives + existing visx/d3. Signal-processing charts (spectrogram, waterfall) rely on hand-computed pseudo-STFT — don't try to implement real FFT. Eye diagram / constellation diagram use seeded LCG for overlaid samples. Flame graph uses `d3-hierarchy` (already installed).
- **Known trickiness:**
  - **Ichimoku** — 5 overlay lines + a shaded kumo region + candlesticks = very dense. At tileSize M (the placeholder), it should just barely fit; consider letting Senkou spans render as thinner lines and the kumo as a 30% alpha fill. Projecting Senkou 26 forward means the last 26 candles on the x-axis have no candlestick but have kumo — this is the chart's canonical feature.
  - **Three-Line Break** — reversal rule is "break the high/low of the previous 3 bars in that direction" — not "break the current trend by one bar". Implement with a 3-bar lookback window.
  - **Eye diagram** — overlay ~200 traces. Each trace is one or two UI (unit-interval) of a synthetic PAM-2 / NRZ signal with hand-computed ISI + noise. Use `path.d` with many subpaths and low-alpha stroke.
  - **Constellation diagram** — 16-QAM has 16 ideal positions in a square grid (4×4). Each ideal point has ~10 noisy received samples around it. Use seeded LCG for the noise.
  - **Lineweaver-Burk** — the data points come from the Michaelis-Menten equation v = Vmax[S]/(Km+[S]). Pre-compute (1/[S], 1/v) for 7 substrate concentrations. The linear fit is exact since LB is a linearisation.
  - **Lewis structure** — a single molecule rendering. Use hand-placed atoms and bonds on a SVG canvas. Dots for lone pairs; single/double lines for bonds. Keep scope to 1-2 molecules.
  - **Phase portrait** — vector field on a grid (arrows pre-scaled), plus 4-6 Runge-Kutta-integrated trajectories for the damped oscillator ẍ + 0.2ẋ + x = 0. Integrate for ~200 steps per trajectory with timestep 0.05.
  - **Poincaré map** — easier option: standard map x_{n+1} = x_n + K sin(2π y_n), y_{n+1} = y_n + x_{n+1} (on a torus). Iterate for ~500 points per initial condition, show ~8 initial conditions to reveal the KAM tori + chaotic zone structure. Simpler than Lorenz Poincaré section; no ODE integration needed.
  - **Hubble diagram** — pre-tabulate Hubble's 1929 data (24 galaxies — easy to look up; OK to approximate). Best-fit slope ~500 km/s/Mpc for Hubble's original; also show modern extension with simulated supernova points at larger distances giving a ~70 km/s/Mpc slope.
  - **Dalitz plot** — Dalitz boundary in (m₁₂², m₂₃²) space is a curve derived from kinematic limits. Pre-compute boundary for a specific decay (e.g. K⁰ → π⁺π⁻π⁰ with masses mK ≈ 498 MeV, mπ ≈ 140 MeV). Fill interior with scatter points; add ~2 resonance bands as density enhancements along lines of constant m₁₂² or m₂₃² corresponding to known intermediate resonances.
  - **Architecture diagram** — Manhattan-routed arrows between boxes. Different shapes for service / database / external / queue. Label arrows with protocol + data. Keep scope to ~8-10 boxes; too many boxes and the L tile becomes unreadable.
  - **Flame graph** — `d3-hierarchy` gives the tree layout; rectangles are computed per node with width = proportion of time, x-position = cumulative offset of siblings. Root at bottom (NOT top — that's icicle).
  - **Circuit diagram** — hand-draw each component symbol once (resistor zigzag SVG path, capacitor two-plates SVG, transistor assembly). Use `<defs>` + `<use href>` for repeated symbols. Wire routing strictly Manhattan; dots at 3+ junctions only.
  - **Ladder diagram** — two tall vertical rails as `<line>` elements; rungs as horizontal `<line>` at specific y levels. Contact/coil symbols between rails: `| |` open contact (two vertical lines with small gap), `|/|` closed (add diagonal), `( )` open coil (small circle/ellipse). Rails are powered — each rung evaluates L→R.
  - **Spectrogram** — 2D grid of cells. x = time bin, y = frequency bin, cell fill = power. Hand-compute a pseudo-STFT: for each time slice, evaluate the signal at that time + assign power to the frequencies present (chirp: one frequency per time slice that moves linearly; burst: one time-frequency blob). Use `scaleSequential(interpolateViridis)` from d3-scale-chromatic — WAIT, check if d3-scale-chromatic is installed; if not, hand-roll the colour stops. (Check package.json; if absent, flag and use a hand-coded 5-stop colour scale.)
  - **Waterfall signal** — 12-15 1D line traces offset vertically, each showing amplitude-vs-frequency at a successive time. Hidden-surface removal: sort by depth, fill each trace with background-colour polygon below it so later (lower) traces don't show through earlier (higher) ones.
  - **Influence diagram** — 3 node shapes (rect = decision, ellipse = chance, hexagon/diamond = value). Arrows with filled arrowheads (relevance) + optional dashed arrows (information) — skip dashed for simplicity if rendering at M tile.
  - **Causal-loop diagram** — nodes as text in small boxes or just label text; directed curved arrows with + or − at the arrowhead. Loop label (R1, B1) at the centre of each loop with a small circular rotation-direction arrow.
  - **Geographical heatmap** — simplified region outline (e.g. rough US outline as a `<path>`) + density overlay (hexbin-style or kernel-density blobs at hand-picked (lat, lon) hotspots). Don't try to use real GeoJSON — keep it iconic.
  - **Topographic map** — hand-compute contour lines of a synthetic elevation function (e.g., z = sin(x)·cos(y) + noise; or sum of 3 Gaussian peaks). Draw contour polygons at levels 100, 200, 300, 400m. Add spot-height labels at the 3 peaks. Hachure ticks on downslope side of each contour.
