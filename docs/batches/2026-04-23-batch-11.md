## Notes

- **Deps likely needed:** none. All charts use SVG primitives + existing `d3-hierarchy` + `d3-shape`. Vector-field/streamline use hand-rolled RK4 integration. Recurrence plot is a heatmap. Piper/Stiff diagrams use triangle/polygon primitives. Exploded-view and P&ID are hand-placed geometry.
- **Known trickiness:**
  - **Bifurcation diagram:** for each r iterate the logistic map 1000 times, discard first 500, plot 500. For ~300 r-values that's ~150,000 iterations at render — well within budget but use `useMemo`.
  - **Recurrence plot:** N=200 gives a 200×200 matrix = 40,000 cells. Render as `<rect>` per recurrent cell (only the 10-15% that exceed threshold) or use `<image>` with canvas-encoded bitmap. Prefer sparse `<rect>` for anchor simplicity.
  - **Piper diagram** — the diamond at top is a rhombus (45° rotation of a square); compositions project there by adding cation-triangle and anion-triangle positions. Three panels share the same sample points — each sample is plotted 3 times.
  - **Stiff diagram** — the kite vertices are at fixed angles around a central vertical axis, x-offsets proportional to concentration. Three cation values on the left, three anion values on the right.
  - **Stereonet** — equal-angle (Wulff) great circles are NOT straight lines in 2D; they're circular arcs. Use the projection formulas: for a plane with dip direction φ and dip angle δ, its great-circle arc projects to a specific arc of the net. Look up Allmendinger's *Structural Geology Algorithms* Ch. 3 for formulas, or hand-approximate for a few angles.
  - **Galbraith plot** — not a polar plot proper but a radial-line plot; x is 1/σ (precision), y is standardised z-score; the "angle from origin" encodes the age estimate via an angular scale on the arc.
  - **Equivolume chart** — each box's width is proportional to that session's volume. Plot on a cumulative-volume x-axis.
  - **Market profile** — the TPO letters (A, B, C, …) at each price row. Letter layout: for each 30-min period k and each price p traded in that period, place letter k at position (rowY(p), slotX(p, k)). Letters stack horizontally until price row runs out of width, then wrap.
  - **Ternary contour** — interior isopleths in a ternary plot. Generate z values on a triangular grid, then use a ternary-aware marching-triangles algorithm. Or approximate by generating iso-composition points at specific z values.
  - **Potential-energy diagram** — a single smooth curve using a hand-chosen parametric Gaussian hump + sigmoid plateaus. Add the catalysed dashed curve at lower Eₐ.
  - **Vector field** — arrows at a grid of points. Each arrow is a line + arrowhead polygon. Normalise arrow length (constant visible length) and encode magnitude via line thickness OR separate colour-key — NOT colour alone.
  - **Streamline plot** — Runge-Kutta-4 integration of the same field. Start from hand-picked seed points; integrate for ~100 steps in each direction.
  - **P&ID** — many symbols. Tank = open-top rectangle. Pump = circle with a triangle inside. Control valve = two triangles meeting at a point, arrow line through them. Heat exchanger = circle with two zigzag lines across. Instrument = circle with 3-letter tag inside. Keep scope to ~6 equipment items.
  - **Exploded-view** — pick a bicycle hub or a pen. Components separated along one axis with dashed centre-lines showing assembly path. Leader lines with numbered callouts.
  - **Stock-and-flow** — stocks as rectangles. Flows as double-line pipe arrows with a "valve" symbol (butterfly shape or clover) in the middle — this is the rate-controller. Auxiliaries as circles. Information signals as thin arrows (no valve).
  - **Concept map** — nodes as rounded rectangles with concept labels. Edges as arrows with relationship-label text on the edge midpoint ("is a", "causes", "requires"). Top-down layout with 2-3 levels.
  - **Radial tree** — `d3-hierarchy.cluster().size([2*Math.PI, radius])`. Convert polar→cartesian. Draw links as arcs or straight radial lines.
  - **Packed circle** — `d3-hierarchy.pack().size([iw, ih]).padding(3)`. Draw each node as a `<circle>` with `r = d.r`, `cx = d.x`, `cy = d.y`. Label leaves only.
  - **Dot density map** — simplified US outline as a `<path>`. Per-state population → count of dots. Distribute dots uniformly within the state polygon using rejection sampling (pick random point in bounding box, keep if inside polygon).
  - **Proportional symbol map** — world-map outline (extremely simplified; no need for real topojson). Circles at major city coordinates, radius = √(population / maxPop) × maxRadius.
